## Notes

- `IN-AU-STYLE-003` (front-matter/objectives) deliberately stays in the author pack -- book/course IA, not shared prose hygiene. The author style tier is a single rule (a legal numbering gap, 001/002/004/005 migrated out).
- WP02 tier placement: the D4 content-owned list mixed mechanical (page meta, alt-text, link text) and judgment (scannability, CTA, reading-level) rules. Placed each by check-type into style vs craft, matching the author convention; D4 scope unchanged.
- WP03: `critic-prose` resolves its discipline from config `languages` (author, content, or both), so one critic serves every prose discipline (D3). The `--lang` invocation override exists for mixed projects or an unambiguous target.
- WP04: `prose` (the `IN-PR-*` base) is deliberately NOT a `lang init`-able discipline -- it has no `templates/prose/`, so it does not appear in `intent lang list` and cannot be declared as a project language. It is a base other prose disciplines pull in, like `agnostic`.
