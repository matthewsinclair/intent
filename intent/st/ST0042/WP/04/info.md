# WP-04: Fix AGENTS.md generation paths and sections

## Objective

Make `intent agents sync` generation truthful and self-consistent (theme T4): read installed skills/agents from where the installers actually write, make the `_default` template pass the tool's own validator, and resolve the template-vs-heredoc Highlander split.

## Evidence

- F-PLG-3 (MEDIUM, confirmed): the generator reads `$PROJECT_ROOT/.claude/{skills,agents}` while installers write to `$HOME/.claude/...`, so "Installed Skills" reads "No skills installed" on a fully provisioned machine (live `AGENTS.md:66`; generator `intent/plugins/agents/bin/intent_agents:74,102`).
- F-TPL-10 (MEDIUM, reported): `templates/_default/AGENTS.md` lacks three of the four sections `intent agents validate` requires, so a fresh `--template _default` install fails the tool's own validator; it also self-describes as "auto-generated by intent agents sync" though sync replaces it with divergent heredoc content -- a Highlander split between template and heredoc (`intent_agents:275-512,545,578-593`).

## Deliverables

- Generator reads `$HOME/.claude/{skills,agents}` (or wherever the installers write -- verify against `intent claude skills install`).
- `_default` template carries all validator-required sections; a fresh `--template _default` install passes `intent agents validate`.
- One source of generated content (template or heredoc, not both diverging) per the single-template-source project rule.
- Regression tests covering the generator read-path and the validate round-trip.

## Acceptance Criteria

- [ ] `intent agents sync` on a provisioned machine lists the actually-installed skills/agents.
- [ ] `intent agents init --template _default && intent agents validate` passes.
- [ ] No divergent template/heredoc pair remains for AGENTS.md content.
- [ ] Full bats suite green; root AGENTS.md regenerated via `intent agents sync`.

## Dependencies

- WP-09 part A (fake-HOME helper) first -- generator tests touch `$HOME`.
