# v2.16.0 -- the content pack, and the prose base beneath it

Minor. Adds the **`content` (web-content) project-type pack** and the **`IN-PR-*` shared prose base** it stands on. Opt-in; zero behaviour change until a project runs `intent lang init content`.

## The shape

`author` (v2.15.0) was the first non-code discipline. `content` is the second. The obvious way to build it -- copy the author pack and swap the rules -- would have been a textbook Highlander violation: two divergent copies of banned-filler, the trope pass, heading hygiene, drifting apart within a quarter. So ST0053 is half refactor, half greenfield.

**The refactor (WP01).** The mechanical prose-hygiene rules that any prose discipline shares -- banned filler and `eg`-not-`e.g.`, no vanity metrics, heading hygiene, the mechanical trope pass -- lifted out of `author` into a new `IN-PR-*` prose base (`rules/prose/`). The four rules moved with migration aliases; the `author` pack dropped to its genuinely discipline-specific rules (front-matter / objectives + the four craft rules). A prose project now stands on the base plus its discipline pack, one copy of the shared surface.

**The greenfield (WP02).** `content` adds six web-distinct rules: `style` (mechanical) -- page meta present (title / description / canonical), image alt-text, descriptive link text; `craft` (judgment) -- scannability and web voice, one clear primary call to action, reading level. The generic prose hygiene it inherits from the base; it re-owns none of it.

**One critic (WP03).** `critic-author` became `critic-prose`: a single critic parameterised by the declared language. It loads the `IN-PR-*` base plus whichever of `author` / `content` the project declares (both, if both), resolved from config `languages`. The two-form detrope survives, re-anchored to the base's `IN-PR-STYLE-004`.

**Canon, skill, dispatch (WP04-05).** `intent lang init content` installs the content canon (allowlist-free -- `templates/content/` is all it took). `/in-content-essentials` carries the content pipeline; `/in-review` and `/in-session` route `content -> critic-prose`. The `/in-review` prose-project note and the D7 no-code-critic exclusion generalised from "author-only" to "prose-only".

## The two new language codes

`PR` (prose) for the base, `CO` (content) for the pack -- each threaded through the same five lang-code sites the `AU` code taught us in ST0052 (rule-schema enum, id-scheme, index-generator regex, the validator, and `LANG_SUBDIRS`), each guarded.

## Deferred

The headless prose gate (D4) stays deferred, as in v2.15.0. The WP06 self-dogfood confirmed the reason again: the content pack's own prose is clean, but the mechanical grep false-positives on rule files (which quote antipatterns deliberately) and fenced examples -- so a headless gate still needs a confirmation / house-style suppression layer before it can drive a pre-commit stop. `critic-prose` is on-demand (`Task`) only.
