# v2.14.0 -- shipped 2026-07-02

Two threads, one release: `intent todo` (ST0050) and a generated-file width fix (ST0051). A minor -- new command surface -- with no migration.

## intent todo (ST0050)

A flat DOING / TODO / DONE view of `intent/st/**`, projected from real `status:` so it cannot drift. Built out from a Lamplight prototype (captured as ST0050) into first-class, tested Intent across six work packages:

- **WP-01 read path** -- minimal `todo.md` (bare bucket headings + data, no boilerplate) + `--json` keyed-by-bucket export sharing one enumeration (Highlander).
- **WP-02 verbs** -- `done` / `notdone` / `toggle` wrap `intent st` / `wp`, inheriting the ST0048 close-gate; never hand-edit a checkbox. `parse_wp_specifier` extracted to `intent_helpers`.
- **WP-03/04** -- curated `intent help` entry over the existing fall-through dispatch; the `intent_todo.bats` harness.
- **WP-05** -- README + usage-rules + CHANGELOG + release notes.
- **WP-06 DONE lifecycle** -- the `## DONE:<T>` flush watermark; `done --flush` / `--prune`; `completed:` upgraded to an ISO 8601 timestamp for exact flush, with legacy `%Y%m%d` tolerated on every read path.

Dogfooded on itself: ST0050 closed through its own gate at 23/23, and `intent st done` stamped it as the first ISO `completed:`.

## Width fix (ST0051)

`intent st sync --write` hard-coded 80 columns, clipping the `steel_threads.md` slug column. Now: generated files size to a new `dft_width` config field (default 120); stdout keeps the terminal width; `--width N` overrides both. A `get_default_width` helper single-sources it.

## Rulings

- Sticky watermark, not auto-daily: `<T>` is authoritative in the heading; `update` preserves it, only flush advances it. "Swept daily" became "swept on flush".
- Verb placement: `done --flush` / `done --prune` (flags on `done`), honouring the original phrasing; disambiguated from `done <spec>` by argument.
