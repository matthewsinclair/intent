# v2.15.0 -- shipped 2026-07-03

The `author` project-type pack (ST0052): the first non-code discipline on Intent's `languages` axis. A project that declares `languages: [author]` gets an authoring rule pack, a prose critic, canon templates, and an essentials skill -- activated exactly the way a code language is. A minor -- new project-type surface -- strictly opt-in, with no migration and zero behaviour change for projects that do not adopt it.

## The pack (ST0052)

Built across six work packages on the Intent whiteboard:

- **WP01 -- the `AU` code.** `author` admitted as a first-class rule-library language across the rule-id validator, the schema, and the canon enumerator (`LANG_SUBDIRS`). A discovery correction along the way: validating by path was not enough -- the enumerator gates `list` / `index`, and it lived in `rules_lib.sh`, not where `id-scheme.md` had pointed.
- **WP02 -- nine rules, two tiers.** `style` (mechanical, greppable) and `craft` (judgment, critic-as-reader). The mechanical trope pass references the single `in-detrope` trope catalogue rather than forking a new indicator set (Highlander; D5).
- **WP03 -- `critic-author`.** The first non-code critic. Two modes (`review` mechanical, `craft-check` judgment) and two-form detrope: mechanical by default, the full `/in-detrope` as an on-instruction handoff it recommends but never runs.
- **WP04 -- lang canon.** `intent lang init author` installs `RULES-author.md` + `ARCHITECTURE-author.md` and writes config `languages`, no `intent lang` change needed -- the config/template layer was already allowlist-free (D2).
- **WP05 -- skill + wiring.** `/in-author-essentials`, and the `author -> critic-author` dispatch in `/in-review` + `/in-session`. An author-only project runs no code critic; a mixed project runs both, each on its own subtree (D7).
- **WP06 -- dogfood + docs.** The mechanical tier run by hand against the pack's own prose.

## Rulings

- **Two tiers, not one.** Mechanical (greppable, default) vs judgment (critic-as-reader, on instruction) -- D3. The dogfood proved why: run against real prose, every mechanical hit was a house-style false positive that only the judgment step could dismiss.
- **Headless prose gate deferred (D4).** `.md` alone cannot route a file to the author pack, and Intent's own `--` house style trips the trope catalogue's dash-overuse regex -- a headless gate would be swamped without a confirmation layer. `critic-author` is on-demand only.
- **Reuse the axis, not a new one.** `author` rides the existing `languages` mechanism (D1), so a courseware repo can be `languages: [elixir, author]` and load both packs.

## Close

ST0052 closed through its own acceptance gate; the ST-level sign-off was on the manual-dogfood approach plus the WP05 wiring tests. The `content` (web-content) pack is deferred -- WP02-WP05 built the shareable pattern it will copy. Built on the Intent whiteboard (hv + cc); cc drove, hv adjudicated and signed off.
