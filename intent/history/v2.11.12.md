# v2.11.12 — shipped 2026-06-11

Tag `v2.11.12` (commit `574b015`) pushed to both remotes; GitHub release published at https://github.com/matthewsinclair/intent/releases/tag/v2.11.12. Patch shipping the full ST0042 arc — the Fable 5 architectural review of Intent, all nine work packages executed in a single session arc — plus the ST0041 MFIC harvest. ST0042 ran as MFIC exercise #1: every fix carried a red-phase proof against HEAD and a regression test or mechanical guard.

What landed, in execution order:

- **WP-09a test HOME isolation** — the bats suite no longer reads or writes the real `$HOME`; fake-home plumbing in test helpers.
- **WP-01 config-eval RCE eliminated** — `eval`-based config parsing replaced with `read_config_field` jq reads. Untrusted `config.json` content can no longer execute.
- **WP-05a `canonical_status`** — one synonym table for steel-thread status normalisation; the divergent per-call-site mappings deleted.
- **WP-03 silent successes** — four paths that reported success without doing the work now fail loudly via `error()`.
- **WP-04 AGENTS.md generation** — generator reads `$HOME/.claude` correctly; generated content matches the install reality.
- **WP-05b Highlander consolidation** — `get_intent_version` fallback, config parsing, `find_project_root`, new `resolve_st_dir`, and `ext_root_dir`/`ext_enumerate_names` all single-sourced in `bin/intent_helpers` and `export -f`'d; duplicate definitions across `intent_config`, plugin libs, and scripts removed.
- **WP-02 rules-path drift finished** — nine skills' SKILL.md plus templates and heredocs now route rule access through `intent claude rules list/show`; a mechanical guard test (`tests/unit/rules_path_guard.bats`) pins the class shut. Closes the follow-up flagged in v2.11.11.
- **WP-07 MODULES.md reconciliation** — registry rows corrected, Advisory Family table added, and the `intent modules check` stale-checker now honours `file::function` rows (it previously mis-flagged them).
- **WP-08 canon docs + `intent st cancel`** — canon docs (usage-rules.md, working-with-llms.md, critics.md, rules.md, README, writing-extensions.md, pre-commit-hook.md) reconciled with as-built behaviour; new `intent st cancel` subcommand mirrors `done` with Cancelled status + CANCELLED relocation.
- **WP-06 prune** — `bin/intent_main`, `bin/intent_minimal`, `bin/stp`, and `intent audit` retired (Highlander: critics are the canonical review engine; credo templates survive via `intent st zero`); orphan template sets (`lib/templates/eng/`, `lib/templates/usr/`) deleted. Upgrade-subsystem dead scaffolding deliberately excluded — ST0043 owns those deletions.
- **WP-09b vacuous-test rewrites** — critic-pipeline tests rewritten against real runner behaviour using synthetic rules and stub-intent hooks; first behavioural coverage for untested modules. Coverage-as-falsification immediately exposed two commands dead for multiple releases, both fixed in passing: `intent organize` dispatch (script renamed from `intent_organise`) and `intent llm usage_rules` (dead root-canon path).

ST0041 (MFIC exploration) closed alongside: adopted practices — red-phase against HEAD, mechanical-guard-per-defect-class, untested-module-as-unfalsified-claim, gate-integrity maintenance, vacuous-test detection; deferred pending field evidence — CI gate promotion, cross-session test authorship, blocking authority. Harvest in `intent/st/COMPLETED/ST0041/impl.md`; the execution-phase leak write-up in `intent/st/COMPLETED/ST0042/impl.md`.

Bookkeeping that rode the release: done-work archive split (`intent/done.md` terse ledger + `intent/history/v2.11.*.md` verbose narratives, wip.md slimmed); `intent st new` version-stamp fix (`f359917`).
