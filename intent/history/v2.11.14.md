# v2.11.14 — shipped 2026-06-14

Tag `v2.11.14` (commit `60dd82a`) pushed to both remotes; GitHub release published at https://github.com/matthewsinclair/intent/releases/tag/v2.11.14. Patch fixing `intent organize` on Linux.

## The bug

`intent organize` tallied moves with `((moved_count++))` / `((kept_count++))` under `set -e`. In bash, `((x++))` is a post-increment: it evaluates to the value _before_ the increment, and `(( ))` returns exit status 1 when its result is 0. So the first increment, with the counter at 0, returns 1 -- and bash 5.x's `set -e` aborts the script right there, after printing the first move but before the summary. macOS bash 3.2 does not act on it in that loop/case context, so the suite was green on macOS (locally and in CI) while Ubuntu CI went red the moment `intent organize` was resurrected in v2.11.12. The macOS-only-green signal masked a genuine product break -- `intent organize` exited 1 on every modern-bash Linux, moving only the first thread before dying -- for four releases.

## The fix

The whole defect class is converted to `x=$((x + 1))`, an assignment, which always returns 0 so `set -e` has nothing to act on -- on any bash. Six sites: three in `bin/intent_organize` and three latent same-class sites in `bin/intent_helpers` (the frontmatter parser's `((line_num++))`, which starts at 0 every call, and two `[ -f ] && ((count++))` legacy-config tallies). A mechanical guard (`tests/unit/set_e_increment_guard.bats`) greps `bin/` and `scripts/` for naked `((x++))` / `((x--))` and fails on any hit, pinning the class shut. The existing `organize_commands.bats` tests, already red on Ubuntu, are the behavioural regression.

## Why it hid

CI runs on both macOS and Ubuntu, but the local pre-release suite (and the release pre-flight's test run) is macOS only -- where the bug is silent. The Ubuntu job had been red since v2.11.12, but a red Ubuntu leg behind a green macOS leg and a green local run did not block the v2.11.12 or v2.11.13 releases. The lesson banked: a per-platform CI split needs the failing leg surfaced, not just the aggregate.
