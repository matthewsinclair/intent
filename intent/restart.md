The user noticed that the v2.10.x rewrite of the agents post described "language detection runs against the project root and pulls in whatever language-specific essentials apply" — and called this out as the **opposite** of the design intent. User picked **Option B** (explicit `languages: []` field in `config.json`) and said "I want this fixed PROPERLY, no half-measures."

## Lessons from this session

- **Detrope discipline scales when applied per-pass.** Three passes per blog draft, with audit after each, eliminates most AI tells. The patterns to watch are bold-first bullets, "X, not Y" / "X rather than Y" forms, three-parallel-verb tricolons, symmetric framing pairs, slogan closers, and magic adverbs. Each pass catches what the previous one introduced.

- **Drafts disagree with intent when the underlying mechanism is wrong.** The agents post described language probing because that's what the SKILL.md does. The user noticed the prose felt off, which surfaced the deeper regression. When prose feels off in a doc, sometimes the bug is one layer down in the system being described, not in the prose.

- **Idempotence is a property of probes too.** The AGENTS.md version probe was greedy and picked the first match in the file. Once content references like `migration-v2.10.0` lived above the generation footer, the probe started reading the wrong version every run. Anchor probes to specific markers, not generic patterns.

## Risks for post-compact

- The schema migration from v2.10.x to v2.11.0 adds a `languages: []` field. Existing fleet projects need back-fill; the migration should infer from `intent/llm/RULES-*.md` presence, not require user action.
- BATS test rework on the probe sites: `in_session_skill.bats`, `pre_commit_hook.bats`, `intent_lang.bats`, and possibly review-dispatch tests. Probe assertions out, config-read assertions in.
- Polyglot projects: array order is the explicit declaration; first entry is the primary where a primary is needed. Document this in the design before coding.
- The pre-commit hook template runs in a fresh shell with `set -e`. Reading `intent/.config/config.json` requires `get_config_field` from `intent_helpers` (already exists) — don't re-implement JSON parsing in the hook.
